Cara Menjalankan Project

  1. Clone repository dari GitHub ke komputer lokal Anda.
  2. Install semua dependencies yang dibutuhkan:
    npm install
  3. Jalankan local server Node.js:
    node server.js
Catatan Penting: Pastikan port 3000 tidak sedang digunakan oleh aplikasi atau server lain. Aplikasi otomatis berjalan di http://localhost:3000.

Manajemen Database

Project ini menggunakan konsep Data Dummy JSON (Tanpa database relational SQL seperti MySQL).

Seluruh data mentah disimpan dengan aman di dalam direktori src/data dalam format file ekstensi JavaScript / JSON.

Alur Request (Lifecycle)

  1. Client mengirim request via aplikasi testing seperti Postman atau browser.
  2. Request diterima oleh sistem routing utama (Router).
  3. Router meneruskan request melewati Middleware (Validasi & API Key).
  4. Request diproses di dalam fungsi bisnis Controller.
  5. Data hasil olahan dikembalikan lagi ke Client sebagai Response JSON.

Format Dokumentasi & Skenario Respon

API secara konsisten mengembalikan data keluaran dengan format standard objek JSON berikut:

{
  "status": "success",
  "message": "data berhasil diambil",
  "data": []
}

HTTP Status Code

  • 200 / 201 : Success & Created
  • 400 : Bad Request (Input salah)
  • 401 : Unauthorized (API Key salah)
  • 404 : Not Found (Data kosong)
  • 500 : Internal Server Error

Kesimpulan & Analisa Akhir

RESTful API Toko Buku ini memudahkan pengelolaan data buku dan transaksi secara efisien menggunakan Node.js dan Express.js. Sistem dirancang bersifat modular, sangat mudah dikembangkan, dan siap diintegrasikan dengan database riil (seperti MySQL/PostgreSQL) di masa mendatang.
