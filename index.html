Cara Menjalankan Project

  1. Clone repository dari GitHub
  2. Install dependencies dengan npm install
  3. Jalankan server dengan node server.js
  4. Server akan berjalan di localhost:3000

Buat Database

Project ini menggunakan data dummy JSON (tanpa database SQL).

Data disimpan pada folder src/data dalam bentuk file JavaScript/JSON.

Jalankan Server

node server.js

Pastikan port 3000 tidak digunakan aplikasi lain.

Catatan Penting Penggunaan :id

Parameter :id digunakan untuk mengambil data spesifik berdasarkan ID.

Contoh:

GET /api/v1/buku/1
PUT /api/v1/buku/2
DELETE /api/v1/buku/3

Alur Request (Lifecycle)

  1. Client mengirim request (Postman / browser)
  2. Request masuk ke router
  3. Router meneruskan ke controller
  4. Controller memproses data
  5. Response dikirim kembali ke client

Daftar Endpoint

Endpoint tersedia untuk Buku dan Pembelian (CRUD lengkap).

GET /api/v1/buku
GET /api/v1/buku/:id
POST /api/v1/buku
PUT /api/v1/buku/:id
DELETE /api/v1/buku/:id
GET /api/v1/pembelian
GET /api/v1/pembelian/:id
POST /api/v1/pembelian
PUT /api/v1/pembelian/:id
DELETE /api/v1/pembelian/:id

Panduan CRUD & API Key (Postman)

  • Gunakan Postman untuk testing API
  • Method GET untuk membaca data
  • Method POST untuk menambah data
  • Method PUT untuk mengubah data
  • Method DELETE untuk menghapus data
  • Jika ada API Key, tambahkan di Headers

Dokumentasi API & Skenario Respon

API akan mengembalikan response dalam format JSON.

{
  "status": "success",
  "message": "data berhasil diambil",
  "data": []
}

Middleware

Middleware digunakan untuk:

  • Validasi request
  • Logging
  • Autentikasi API Key

Status Code HTTP

  • 200 = Success
  • 201 = Created
  • 400 = Bad Request
  • 401 = Unauthorized
  • 404 = Not Found
  • 500 = Server Error

Kesimpulan & Analisa Akhir

RESTful API Toko Buku ini memudahkan pengelolaan data buku dan transaksi secara efisien menggunakan Node.js dan Express.js. Sistem ini bersifat modular, mudah dikembangkan, dan dapat diintegrasikan dengan database nyata di masa depan.
